 1. index.html 을 더블클릭 (Chrome 또는 Edge 권장)
 2. [▶ 플레이] -> 곡 선택 (Diamond In the Dark)
 3. 난이도 선택 (EASY / NORMAL / HARD, 처음이면 EASY)
 -> 바로 시작됩니다! (곡이 게임에 내장되어 있어 따로 파일 안 불러와도 됨)

 ※ index.html 과 songs.js 는 같은 폴더에 함께 있어야 합니다.

[채보 직접 만들기]
 [✎ 채보 에디터]
  - "곡 불러오기" 후  [⚡ 음파 분석 -> 채보 자동생성]  으로 초안 생성
  - 도구: 선택/이동, +일반, +프레스(드래그로 길이), 🎨 컬러군
  - 🎨 컬러군: 연타 구간을 드래그해 묶으면, 그 구간은
              매 판 색이 바뀌어도 "한 가지 색"으로 통일됩니다
  - 휠=스크롤, Ctrl+휠=확대, 노트 우클릭=삭제
  - [저장](브라우저 보관) / [내보내기](.json 파일)
  - 만든 채보는 플레이 화면의 "내 파일로 직접 플레이"에서 불러올 수 있어요

[★ 내 채보는 어디 저장되나요? (중요)]
 - 내가 만든 곡/채보는 index.html·songs.js 파일 안이 아니라
   "브라우저 저장소"에 저장됩니다.
 - 그래서 업데이트 방법에 따라 결과가 달라요:
    ✅ 같은 폴더에서 index.html(과 songs.js)만 새 파일로 덮어쓰기
       → 경로가 그대로라 내 채보가 그대로 유지됩니다. (권장)
    ⚠️ 압축을 "새 폴더"에 풀어서 실행
       → 브라우저가 다른 위치로 인식해 내 채보가 안 보입니다.
       (예전 폴더엔 남아있음)
 - 안전장치로 백업 기능을 넣었습니다:
    [채보 에디터] → "내 채보 백업 / 복원"
      · [전체 백업 (.json)] : 내 모든 곡+오디오를 파일 하나로 저장
      · [백업 불러오기]     : 그 파일로 어디서든 그대로 복원(병합)
 - 폴더를 옮기거나 새 버전으로 바꾸기 전에 한 번 백업해두면
   브라우저를 바꿔도, 새 폴더에서 실행해도 그대로 살릴 수 있어요.

[★ 수정한 채보를 "다른 사람에게도" 적용하려면 (중요)]
 - 에디터에서 수정한 채보는 "내 브라우저"에만 저장됩니다.
   폴더(index.html, songs.js)엔 안 들어있어요. 그래서:
    · 게임 폴더를 "그냥 압축"해서 보내면 → 수정 반영 안 됨(원본 그대로)
    · 웹페이지는 보안상 폴더 안 파일을 직접 못 고침 → 자동 반영 불가
   => 내 수정을 songs.js 에 한 번 구워넣는 단계가 꼭 필요합니다.

 - [채보 에디터] → "다른 사람에게 배포하기"  (두 가지 방법)

   방법 A (간편, Chrome/Edge):
     1. [📁 게임 폴더에 바로 저장] → 게임 폴더 선택
        → 그 폴더의 songs.js 가 수정본으로 바로 갱신됩니다
     2. 그 폴더를 압축해서 배포 끝
     ※ 더블클릭(file://) 실행에선 막힐 수 있어요. 그땐 방법 B.

   방법 B (어디서나):
     1. [📦 songs.js 파일로 받기] → songs.js 다운로드
     2. 배포 폴더의 songs.js 를 방금 받은 파일로 "교체"
     3. 그 폴더(index.html + 새 songs.js + README.txt)를 압축해 배포

 - 받은 사람은 새 songs.js 가 든 폴더로 실행하면 수정본이 적용됩니다.
   (index.html 은 안 바꿔도 됩니다)

[★★ 자동반영 — 한 번 설정하면, 이후 수정이 "모두에게" 자동 적용]
 매번 재배포하기 싫다면, 채보를 인터넷(GitHub Gist)에 두고 게임이
 실행될 때마다 최신본을 불러오게 할 수 있습니다.
 (플레이어가 인터넷 연결돼 있어야 적용됩니다. 안 되면 내장 채보로 그냥 플레이)

 ── 처음 한 번만 설정 ──────────────────────────────
 1) github.com 가입 → gist.github.com 접속
 2) 파일 이름을 charts.json 으로, 내용은 []  (대괄호 두 개)만 넣고
    "Create public gist" 클릭
 3) 우측 [Raw] 버튼을 눌러 열린 주소를 복사
    (형태: https://gist.githubusercontent.com/내이름/긴아이디/raw/charts.json)
 4) index.html 을 메모장으로 열어서 윗부분의
       const REMOTE_CHARTS_URL = "";
    의 "" 안에 그 주소를 붙여넣고 저장
       예) const REMOTE_CHARTS_URL = "https://gist.../raw/charts.json";
 5) 이 index.html 이 든 폴더를 압축해 사람들에게 "한 번" 배포
    (이 주소가 박힌 버전을 모두가 갖게 하는 것 — 이게 마지막 재배포)

 ── 이후 채보를 바꿀 때마다 (재배포 불필요!) ──────────
 1) [채보 에디터]에서 채보 수정·저장
 2) "온라인 자동반영" → [📤 자동반영용 채보 만들기] 클릭
    (charts.json 내용이 클립보드에 복사됩니다)
 3) 만들어둔 Gist 를 열어 [Edit] → 전체 지우고 붙여넣기 → [Update]
 4) 끝! 인터넷 연결된 사람은 다음 실행 때 자동으로 최신 채보를 봅니다.

 ※ 비워두면("") 자동반영은 꺼지고 평소처럼 내장 채보로 동작합니다.
 ※ 자동반영은 "내장곡(예: Diamond)을 수정한 채보"에 적용됩니다.
   완전히 새 곡(내 음원)은 오디오도 호스팅해야 해서 별도 작업이 필요해요.

즐기세요!
